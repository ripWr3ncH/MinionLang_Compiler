Core Components

The building blocks of our playful yet powerful compiler.

Semantic Checks

Robust scope and type validation ensures your Minion code is logically sound before it ever hits the processor.

Lexical Analysis

Powered by Flex for lightning-fast tokenization of your source code.

Syntax Analysis

Bison-driven LALR parsing to build syntax trees with precision.

Execution Proof

We cross-reference MinionLang execution with C output to verify translation fidelity.

IR Generation

3-address intermediate representation for optimization and portability.

Constant Folding

Compile-time arithmetic evaluation to prune binary size.

Quick Answers

Difference between lexer and parser?expand_more
The lexer converts characters into tokens. The parser validates token sequence against grammar rules.
Why semantic analysis is necessary?expand_more
It catches meaning-level issues such as undeclared identifiers and type mismatches.
What is constant folding?expand_more
Constant expressions are computed during compilation instead of runtime.
How scope handling works?expand_more
A nested scope stack controls symbol visibility and supports safe shadowing.
